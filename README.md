# 과제 #2 

## 주제: top ps, jobs, kill 명령어 조사 

#

## 1.명령어 한눈 정리
| 명령어 | 역할 | 주요 기능 / 옵션 | 예제 |
| ------ | --------------- | ------------ | -------------------------------- |
| `top` | 실시간 시스템 모니터링 | CPU, 메모리, 프로세스 상태 확인 / 정렬: `P`(CPU), `M`(메모리) | `top → P` (CPU 순 정렬) |
| `ps` | 현재 프로세스 스냅샷 | 전체 프로세스 보기: `ps -ef` / 사용자별 보기: `ps -u 사용자명` / CPU/메모리 정렬: `ps aux --sort=-%cpu` | `ps aux --sort=-%mem` |
| `jobs` | 현재 터미널 작업 관리 | 백그라운드/일시정지 작업 확인: `jobs` / 포그라운드: `fg %번호` / 백그라운드: `bg %번호` | `sleep 100 &` → `jobs` → `fg %1` |
| `kill` | 프로세스 종료 / 신호 전송 | 종료: `kill PID` / 강제 종료: `kill -9 PID` / 일시정지: `kill -STOP PID` | `kill 1234` / `kill -9 1234` |


#

## 2-1 top 명령어란

### 1. top 명령어란?
- **실시간 시스템 모니터링 도구**예요.
- CPU, 메모리, 프로세스 상태를 확인할 수 있어요.

### 2. 화면 구성 
`top` 화면은 크게 3부분으로 나눌 수 있어요.
1. 시스템 정보 (상단)
- **uptime** : 시스템 켠 지 얼마나 됐는지
- **tasks** : 전체 프로세스 수
- **load average** : CPU 사용률 평균
2. 프로세스 리스트 (하단)
- **PID** : 프로세스 ID
- **USER** : 프로세스 실행 사용자
- **%CPU** : CPU 사용률
- **%MEM** : 메모리 사용률
- **COMMAND** : 실행한 프로그램 이름
- **%CPU, %MEM** : CPU와 메모리 사용률

### 3. 자주 쓰는 단추키
| 조작 키 | 기능 | 설명 |
| ---- | ------------ | ------------------------- |
| `P` | CPU 사용량 순 정렬 | CPU를 많이 쓰는 프로세스가 위쪽에 표시됨 |
| `M` | 메모리 사용량 순 정렬 | 메모리 많이 쓰는 프로세스가 위쪽에 표시됨 |
| `k` | 특정 PID 종료 | 종료할 프로세스의 PID를 입력하여 종료 가능 |
| `q` | 종료 | `top` 종료하고 쉘로 돌아감 |

### 4. 실전 예제
1. CPU 많이 쓰는 프로세스 확인:
- `top → P`
2. 메모리 많이 쓰는 프로세스 확인:
- `top → M`
3. 특정 PID 종료:
- `top → k → PID 입력`

#

## 2-2 ps 명령어

### 1. ps 명령어란?
- **현재 실행 중인 프로세스 목록을 보는 명령어**예요.
- `top`이 실시간 모니터링이라면, `ps`는 순간 상태를 보여주는 스냅샷이라고 생각하면 돼요.
- 기본적으로 자기 터미널에서 실행한 프로세스만 보여줘요.

### 2. 자주 쓰는 옵션
| 명령어 | 설명 |
| ------------------ | -------------------------------------- |
| `ps -e` 또는 `ps -A` | 시스템 전체 프로세스 보기 |
| `ps -f` | 전체 정보(full) 보기 (UID, PID, PPID, CMD 등) |
| `ps -u 사용자명` | 특정 사용자의 프로세스 보기 |
| `ps aux` | 많이 쓰이는 전체 프로세스 보기 방법 |
| `ps -ef` | 다른 형태의 전체 프로세스 보기 |

> !!! `ps aux`와 `ps -ef`는 거의 비슷하지만 출력 형식이 조금 달라요.

### 3. 출력 컬럼 의미 (대표적)

`SER` : 프로세스를 실행한 사용자

`PID` : 프로세스 ID

`%CPU` : CPU 사용률

`%MEM` : 메모리 사용률

`VSZ` : 가상 메모리 사용량 (KB 단위)

`RSS` : 실제 메모리 사용량 (KB 단위)

`TTY` : 터미널 종류

`STAT` : 프로세스 상태 (R: 실행, S: 대기, Z: 좀비 등)

`COMMAND` : 실행한 명령어

### 4. 실전 예제
1. 내가 실행한 프로세스 보기:
- `ps`
2. 모든 사용자 프로세스 보기:
- `ps -ef`
3. 특정 사용자 프로세스 보기:
- `ps -u 사용자명`
1. CPU 사용률 높은 순으로 정렬:
- `ps aux --sort=-%cpu`
2. 메모리 사용률 높은 순으로 정렬:
- `ps aux --sort=-%mem`

## 2-3 jobs 명령어

### 1. jobs 명령어란?
- **현재 터미널에서 실행 중인 백그라운드 작업(프로세스)을 확인하는 명령어**예요.
- `&`를 붙여서 실행한 작업이나, `Ctrl+Z`로 일시정지한 작업을 관리할 때 사용돼요.
- `jobs` 입력하면 현재 터미널에서 관리 중인 작업 목록을 보여줘요.

### 2. 기본 사용법
| 명령어 | 설명 |
| --------- | --------------------- |
| `jobs` | 현재 터미널의 작업 목록 보기 |
| `jobs -l` | PID까지 포함해서 자세히 보기 |
| `jobs -p` | PID만 출력 |
| `fg %번호` | 백그라운드 작업을 포그라운드로 가져오기 |
| `bg %번호` | 중지된 작업을 백그라운드로 실행 |

 > !!! `%번호`는 `jobs` 명령어로 나온 작업 번호를 의미해요.

### 3. 상태 의미

- `Running` : 현재 실행 중
- `Stopped` : 일시정지됨 (Ctrl+Z로 멈춘 상태)
- `Done` : 완료됨
  
### 4. 실전 예제
1. **백그라운드 실행**

`sleep 100 &`

- 뒤에 `&`를 붙이면 **백그라운드에서 실행**돼요.

2. **작업 목록 확인**

`jobs`

`[1]+  Running                 sleep 100 &`


3. 작업 포그라운드로 가져오기

`fg %1`


4. 작업 백그라운드로 보내기

`bg %1`

## 2-4 kill 명령어

### 1. kill 명령어란?
- **프로세스를 종료시키거나 신호를 보내는 명령어**예요.
- 종료뿐만 아니라 **일시정지, 재시작 등** 신호를 보낼 수도 있어요.

 `kill PID`
- `PID`는 종료하고 싶은 프로세스의 **프로세스 ID**예요.

### 2. 자주 쓰는 옵션 / 신호
| 옵션 / 신호 | 설명 |
| ---------------- | ----------------------- |
| `kill PID` | 기본: 종료 요청(SIGTERM, 15번) |
| `kill -9 PID` | 강제 종료(SIGKILL, 9번) |
| `kill -l` | 사용할 수 있는 모든 신호 목록 보기 |
| `kill -STOP PID` | 프로세스 일시정지 |
| `kill -CONT PID` | 일시정지된 프로세스 재개 |

> !!! 일반적으로는 그냥 `kill PID`로 충분하고, 안 될 때 `kill -9 PID`를 사용해요.

### 3. 실전 예제

1. 프로세스 종료

`kill 1234`

- PID가 1234인 프로세스를 정상 종료

2. 강제 종료

`kill -9 1234`

- 프로세스가 종료되지 않을 때 강제로 종료

3. 백그라운드 작업 종료

`jobs`

`kill %1`

- %번호로 jobs의 작업 종료 가능

### 팁
- `ps`, `jobs` 명령어로 PID를 확인하고, 그 다음 `kill`을 쓰면 안전하게 프로세스를 관리할 수 있어요.
- `kill`은 “조용히 부탁하는 종료” → 안 되면 “강제로 꺼버리기” 두 단계로 생각하면 쉬워요.
